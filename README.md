# mspm0kit-skill

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

AI-powered CLI agent skills for the Tianqiaoxing MSPM0G3519 custom development board. One sentence -> complete CCS project with automatic pin mapping, clock tree configuration, peripheral setup, SysConfig CLI + gmake build verification, and DSLite flash.

**English** | [中文](README.zh-CN.md)

---

## What It Does

| Skill | Purpose |
|-------|---------|
| `tianqiaoxing-g3519` | One-sentence CCS project generator: scaffold -> build -> flash. Full board pin table + 10 peripheral reference docs. |
| `mspm0-ccs` | General MSPM0 SysConfig/DriverLib helper. Project inspection, example management, serial console. |

## Quick Start

### 1. Configure paths (first use only)

```powershell
python ~/.claude/skills/tianqiaoxing-g3519/scripts/setup.py
```

Press Enter to accept defaults (CCS `D:\TI\CCS\ccs`, SDK `D:\TI\CCS\mspm0_sdk_2_05_01_00`, XDS110).

### 2. Use in your Agent

```
help me create a UART0 debug console project
```

The skill will:
1. **Think** - identify peripheral, check pin table, confirm clock
2. **Plan** - present what will be created
3. **Code** - run `scaffold.py` to generate the CCS project
4. **Ask** - "Project generated. Compile and test it now?"
5. **Verify** - if you say yes, run SysConfig CLI + gmake and report results

### 3. Import in CCS

File -> Import -> C/C++ -> CCS Projects from `.projectspec` -> Select `<project>.projectspec`

## Supported Features

### Standard Peripherals (from MSPM0 SDK examples)

| Peripheral | SDK Example | Default Pins |
|-----------|-------------|--------------|
| GPIO Output | `gpio_toggle_output` | PB22 |
| UART TX/RX | `uart_rw_multibyte_fifo_poll` | PA10/PA11 |
| UART Console | `uart_tx_console_multibyte_repeated_fifo_dma` | PA10/PA11 (115200 baud) |
| SPI Controller | `spi_controller_multibyte_fifo_poll` | SPI0 on free pins |
| I2C Controller | `i2c_controller_rw_multibyte_fifo_poll` | I2C0 on free pins |
| ADC Single | `adc12_single_conversion` | free ADC pin |
| PWM Timer | `timg_32bit_timer_mode_pwm_edge_sleep` | TIMG0 on free pins |
| Timer Periodic | `tima_timer_mode_periodic_repeat_count` | - |

### Onboard Peripherals (from OLED_UI framework)

```bash
python scripts/scaffold_oled.py my_project --with-imu --with-ws2812 --with-wireless
```

| Module | Driver | Pins | Flag |
|--------|--------|------|------|
| OLED (SSD1312 128x64) | `oledUI/` framework | PA0/PA1 | always included |
| IMU (LSM6DS3) | `hw_lsm6ds3.c` + FusionAhrs | PA28/PA27 | `--with-imu` |
| WS2812 RGB LED | `hw_ws2812.c` + effects | PB26 (TIMA1) | `--with-ws2812` |
| Wireless UART | `mid_wireless_uart.c` | PB17/PB18 | `--with-wireless` |

## Board Pin Table

### Unusable (system-level)

| Pin | Reason |
|-----|--------|
| PA2 | Frequency accuracy control |
| PA5, PA6 | 40 MHz HFXT crystal |
| PA19, PA20 | SWD debug interface |

### Board Peripherals (occupied)

| Pin(s) | Function |
|--------|----------|
| PA0, PA1 | OLED I2C |
| PA10, PA11 | UART0 CH340 USB-C |
| PA27, PA28 | IMU I2C |
| PB6-PB9 | W25Q128 SPI Flash |
| PB17, PB18 | Wireless UART |
| PB21 | ENTER button |
| PB22 | Onboard LED (active-low) |
| PB26 | WS2812 RGB |
| PB27 | Buzzer |

See `skills/tianqiaoxing-g3519/SKILL.md` for the complete reference.

## Directory Structure

```

  SKILL.md                    Agent entry: 4-step workflow + pin table + example index

    serial_console.py         Serial monitor
  tests/                      7 tests (setup, scaffold, build)
```

## Requirements

- CCS Theia with TI Arm Clang 4.0.3 LTS
- MSPM0 SDK 2.05.01.00
- SysConfig 1.24.0
- XDS110 or J-Link debug probe
- Python 3.8+ with `pyserial` (for serial console)
